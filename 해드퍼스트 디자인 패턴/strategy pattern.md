# 전략 패턴
알고리즘군을 정의하고 **캡슐화**해서 각각의 알고리즘군을 수정해서 쓸 수 있게 해 줍니다.
전략 패턴을 사용하면 클라이언트로 부터 알고리즘을 **분리**해서 독립적으로 **변경할 수 있습니다**.


## 낱말 퀴즈
1. 패턴은 _ 애플리케이션을 만드는 데 도움이 됩니다. => FLEXIBLE
2. 패턴을 _에 넣어 주세요! => BRAIN
3. 꽥꽥 소리를 낼 수 없는 오리는? => DECOYDUCK
4. 전략 패턴을 쓰면 코드를 OOO할 수 있습니다. => REUSED
5. 고무 오리는 OOO를 냅니다. => SQUEAK
6. 바뀔 수 있는 부분은 OOO해야 합니다. => ENCAPSULATE
7. 상속보다는 OOO이 더 낫죠. => COMPOSITION
8. 개발에 있어서 바뀌지 않는것! => CHANGE
9. 자바 IO, 네트워킹, 사운드 OOO => APIs
10. 패턴은 대부분 객체지향 OOO을 따릅니다. => PRINCIPLES
11. 베이컨이 들어있는 구운 치즈 샌드위치를 OOO라고 부릅니다. => JACKBENNY
12. 디자인 패턴은 서로 공유하는 OOO입니다. => PRINCIPLES
13. 아까 오창이 OOO패턴을 좋아했죠? => OBSERVER
14. 고수준 라이브러리를 OOO라고 부릅니다. => FRAMEWORKS
15. 다른 사람의 OOO으로 부터 배울 수 있어야 합니다. => EXPERIENCE
16. SimUduck 데모를 선보였던 주주총회가 열렸던 곳의 지명은? => MAUI
17. 시뮬레이터에 있는 문제를 OOO 패턴으로 해결했죠? => STRATEGY
18. 구현 보다는 OOO에 맞춰서 프로그래밍해야 합니다. => INTERFACE
